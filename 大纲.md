# 🎮 校园任务系统（游戏化大世界版）—— 完整功能大纲

---

## 1. 用户系统

- 1.1 注册/登录
  - 身份选择（学生/老师）
  - 学号/工号 + 密码登录
  - **可选功能**：接入全校学号/姓名名单（自动验证身份）
- 1.2 个人主页
  - 头像
  - 昵称
  - 个性签名
  - 身份标识（学生/老师）
  - 等级
  - 信用分
  - **课程表导入**：识别空档期
- 1.3 账户设置（修改密码、绑定手机、注销账号）

---

## 2. 任务系统

- 2.1 任务板块
 - 三个模块（采用大卡片布局）
   - 日常任务（学生发布，需扣除发布者自身积分50%，平台补贴50%）
   - 悬赏任务（老师发布，平台全额补贴，发布者获得奖励积分的50%）
   - 紧急任务（学生/老师均可发布，无平台补贴，发布者全额承担奖励积分）
 - 按距离/类型等筛选后仅显示符合条件的任务卡片

- 2.2 任务发布
  - 预设模板（捎带/咨询/求助等）
  - 填写任务信息（**需支持语言输入转文字**）
    - 标题、描述、图片/附件
    - 任务难度（1-5星）
    - 截止时间
    - 任务地点（地图坐标）
    - 奖励积分
    - 是否紧急
  - 发布限制
    - 学生：日常任务无发布数量限制，需账户积分足够支付发布扣除部分；紧急任务需全额支付奖励积分
    - 老师：悬赏任务每天最多1个，平台全额补贴；紧急任务需全额支付奖励积分
  - AI审核（自动过滤违规内容）
  - 人工申诉（审核不通过时可申诉）

- 2.3 任务领取
  - 查看任务详情
  - 接单条件（等级/信用分门槛，与任务难度挂钩）
  - 单人任务：一旦被接单，进入“进行中”，他人不可接

- 2.4 任务进行
  - 任务状态（进行中/待确认/已完成/已取消/已超时）
  - 任务超时提醒（截止时间前1小时/30分钟）

- 2.5 任务完成
  - 领取者提交完成证明（文字/图片/附件）
  - 发布者确认完成
  - 积分流转
    - 日常任务：发布者扣除奖励积分的50%，平台补贴50% → 领取者获得全额奖励积分
    - 悬赏任务：平台全额补贴 → 领取者获得奖励积分，发布者获得奖励积分的50%
    - 紧急任务：发布者全额支付 → 领取者获得奖励积分
  - 多人任务完成：奖励按参与人数比例分配

- 2.6 任务管理
  - 我的发布（查看/下架/处理完成）
  - 我的接单（进行中/已完成/已取消）
  - 任务历史记录

- 2.7 任务申诉
  - 提交申诉（任务纠纷）
  - 管理员人工处理

- 2.8 任务审核（管理员后台）
  - 查看待审核任务
  - AI审核结果复核
  - 手动通过/驳回

## 3. 积分系统

- 3.1 积分获取
  - 初始积分（新用户注册赠送）
  - 完成任务获得奖励积分
  - 发布任务时平台补贴部分（日常任务/悬赏任务）
  - 系统活动奖励（签到、赛季结算等）
  - 邀请新用户注册
    - 邀请人获得固定积分奖励
    - 根据累计邀请人数，设置阶梯奖励（如：邀请3人额外奖励XX积分，邀请5人额外奖励XX积分）

- 3.2 积分消耗
  - 发布任务时扣除
    - 日常任务：扣除奖励积分的50%
    - 紧急任务：扣除全额奖励积分
    - 悬赏任务：无需扣除
  - 商城兑换道具/实物

- 3.3 积分账户
  - 总积分（可用积分）
  - 冻结积分（已发布但未完成的任务预扣）
  - 积分流水记录（来源/去向/时间/关联任务）

- 3.4 积分上限/下限
  - 积分可为0，不可为负
  - 发布任务时积分不足则无法发布

- 3.5 积分排行榜（见排行榜系统）

- 3.6 赛季积分
  - 每个赛季独立统计
  - 赛季结束后部分积分转化为永久积分（比例可配置）
  - 赛季积分用于赛季排名和奖励

## 4. 成长系统

- 4.1 等级
  - 等级划分（如 Lv.1 ~ Lv.100）
  - 升级条件：完成任务数量 / 累计获得积分（二者综合）
  - 等级特权
    - 奖励积分按比例提升（等级越高，完成任务时获得奖励积分加成越高）
    - 接单大厅推荐权重提升（高等级用户首页推荐高奖励任务的比例更高）

- 4.2 经验值
  - 获取方式
    - 完成任务获得经验
    - 签到/连续签到获得经验
    - 邀请新用户获得经验
  - 等级衰减
    - 长时间不登录，等级逐渐下降
    - 信用分不受等级衰减影响

- 4.3 信用分
  - 初始信用分：100分
  - 扣分规则
    - 被差评：扣分（分值随任务难度变化）
    - 接单后超时未完成：扣分（分值随任务难度变化）
    - 接单后1小时内取消：扣分（分值随任务难度变化）
  - 信用分等级与权限

| 信用分范围 | 可接任务难度 | 可发布任务 |
|:---|:---|:---|
| 100分 | 5个难度全部可接 | ✅ |
| 80-99分 | 4个难度可接（最高难度不可接） | ✅ |
| 60-79分 | 3个难度可接 | ❌ |
| 40-59分 | 2个难度可接 | ❌ |
| 20-39分 | 1个难度可接 | ❌ |
| 0-19分 | 无法接单 | ❌ |
| <40分 | **冻结账户** | ❌（需联系管理员解冻） |

## 5. 社交系统

- 5.1 好友系统
  - 添加/删除好友（需对方同意）
  - 好友列表展示（头像、昵称、当前等级、在线状态）

- 5.2 私信系统
  - 发送/接收私信（接受显示已读，参考抖音私信）
  - 消息列表（按时间倒序）
  - 未读消息提醒（通知系统）
  - 任务相关私信（如任务进度、完成确认等）

- 5.3 任务评价
  - 任务完成后，发布者对领取者进行评价（好评/中评/差评）
  - 评价内容展示在领取者个人主页（可隐藏）
  - 差评影响信用分（见 4.3）
  - 评价与好友状态无关

## 6. 排行榜系统

- 6.1 榜单类型
  - 总积分榜（历史累计积分，含赛季转化部分）
  - 赛季积分榜（当前赛季积分）
  - 等级榜
  - 任务完成榜（按完成任务总数）

- 6.2 榜单周期
  - 总榜（永久）
  - 赛季榜（每学期重置）
  - 周榜（每周一重置）

- 6.3 周榜奖励
  - 周榜前三名获得积分奖励
    - 第一名：奖励 X 积分
    - 第二名：奖励 Y 积分
    - 第三名：奖励 Z 积分
  - 奖励自动发放至账户

- 6.4 榜单展示
  - 全站排名（前100名展示完整信息）
  - 个人排名（用户可查看自己在各榜单中的位置）
  - 榜单更新频率：实时

## 8. 商城系统

- 8.1 商品类型
  - 虚拟道具
    - 双倍经验卡（任务完成后经验翻倍，限时）
    - 积分加成卡（任务完成后积分加成，限时）
    - 免罚卡（抵消一次信用分扣分）
  - 实物奖品
    - 文创产品（校园周边、定制笔记本等）
    - 学习用品（图书、文具等）
    - 电子产品（需高积分兑换）

- 8.2 商品管理
  - 商品上架/下架（管理员）
  - 商品库存管理（实物商品）
  - 商品定价（积分价格）
  - 兑换限制（如：每人每日限购、等级限制）

- 8.3 兑换流程
  - 用户浏览商品 → 选择数量 → 确认兑换
  - 积分扣除 → 生成兑换记录
  - 实物商品：记录收货地址 → 管理员发货 → 用户确认收货
  - 虚拟道具：即时到账（自动发放至用户账户）

- 8.4 兑换记录
  - 用户可查看历史兑换记录
  - 管理员可查看所有兑换记录（用于发货/统计）

## 9. 任务推荐算法

- 9.1 推荐逻辑
  - 一层逻辑
    - 时间维度：根据用户填写的课程表空档期进行推送
    - 空间维度：优先推送步行距离5分钟内的微任务
  - 二层逻辑
    - 基于用户等级
      - 高等级用户在**任务板块**获得高奖励任务的比例更高
    - 基于用户历史行为
      - 根据用户历史完成的任务类型、任务难度、任务地点，推荐相似任务
    - 基于信用分
      - 高信用分用户优先推荐（作为任务推荐排序因子之一）
    - 基于任务热度
      - 接单人数多/浏览多的任务权重提升

- 9.2 推荐场景
  - 任务板块（默认推荐列表）
  - 通知推送（可选，系统向高匹配度用户推送新发布的任务）

- 9.3 推荐过滤
  - 用户已接单的任务不再推荐
  - 用户已拒绝/不感兴趣的任务减少推荐频率
  - 超出用户信用分可接难度范围的任务不推荐

- 9.4 推荐配置（管理员）
  - 推荐权重参数可调（等级权重、历史行为权重、信用分权重等）
  - 推荐算法开关（可切换为简单排序模式）

## 10. 管理员系统

- 10.1 用户管理
  - 查看所有用户列表（学号/工号、昵称、等级、信用分、积分余额、注册时间）
  - 搜索/筛选用户（按身份、等级、信用分区间、注册时间）
  - 查看用户详情（完整信息、任务记录、积分流水、兑换记录）
  - 冻结/解冻账户（冻结后无法登录/接单/发布）
  - 手动调整信用分（需记录操作日志）
  - 手动调整积分（需记录操作日志）

- 10.2 任务管理
  - 查看所有任务列表（发布者、标题、奖励积分、状态、发布时间）
  - 搜索/筛选任务（按类型、状态、难度、时间范围）
  - 任务审核
    - 查看待审核任务
    - AI审核结果复核
    - 手动通过/驳回（驳回需填写原因）
  - 强制下架任务（违规任务）
  - 处理任务申诉（查看申诉详情 → 人工裁决 → 调整积分/信用分）

- 10.3 商品管理（见商城系统 8.2）
  - 商品上架/下架
  - 商品库存管理
  - 商品定价

- 10.4 数据统计与看板
  - 平台总览
    - 总用户数、日活/月活
    - 今日新增任务数、今日完成任务数
    - 今日积分消耗/发放总量
  - 任务统计
    - 任务发布趋势（按日/周/月）
    - 任务完成率
    - 各类任务占比
    - 任务难度分布
  - 用户统计
    - 用户等级分布
    - 用户信用分分布
    - 活跃用户排行（按任务完成数/积分获取数）
  - 积分统计
    - 积分发放来源占比（任务奖励/签到/邀请/赛季结算）
    - 积分消耗去向占比（发布任务/商城兑换）
  - 商城统计
    - 兑换商品排行（按兑换量）
    - 实物商品待发货列表

- 10.5 系统配置
  - 积分参数配置
    - 新用户初始积分
    - 邀请奖励阶梯规则
    - 签到/连续签到奖励规则
    - 赛季积分转化比例
  - 信用分参数配置
    - 各类扣分/加分分值（与任务难度挂钩的配置）
  - 等级参数配置
    - 升级所需经验值曲线
    - 等级衰减规则（多久不登录开始衰减、每次衰减幅度）
  - 推荐算法参数配置（见 9.4）
  - 榜单奖励配置（周榜前三名积分奖励数值）
  - 商城兑换限制配置（每人每日限购、等级限制等）

- 10.6 日志审计
  - 管理员操作日志（谁、何时、做了什么、操作前后数据对比）
  - 系统关键操作日志（任务审核结果、积分调整、信用分调整）
  - 用户违规记录（差评、超时、取消记录）

## 11. 通知系统

- 11.1 通知类型
  - 系统通知
    - 平台公告（管理员发布）
    - 版本更新提醒
    - 赛季结算提醒
  - 任务相关通知
    - 任务发布成功
    - 任务审核结果（通过/驳回）
    - 任务被接单（发布者收到）
    - 任务即将超时（领取者收到）
    - 任务自动撤回（发布者收到）
    - 任务完成确认（发布者收到待确认提醒）
    - 任务完成（领取者收到）
    - 任务申诉结果
  - 社交相关通知
    - 好友申请/通过
    - 私信提醒
    - 动态被点赞/评论
  - 系统奖励通知
    - 周榜奖励发放
    - 签到奖励发放
    - 邀请奖励发放
    - 商城兑换成功/发货通知
  - 信用分变动通知
    - 信用分扣除（说明原因）
    - 信用分低于阈值警告
    - 账户冻结/解冻通知

- 11.2 通知方式
  - 站内消息（消息中心列表）
  - 未读红点提示（菜单栏/消息图标）
  - 可选：邮件通知（绑定邮箱后可选择接收）

- 11.3 消息中心
  - 按时间倒序展示所有通知
  - 按类型筛选（系统/任务/社交/奖励/信用）
  - 标记已读/全部已读
  - 删除通知

- 11.4 通知设置（用户可配置）
  - 各类型通知的开关（开启/关闭）
  - 邮件通知开关（如已绑定邮箱）

- 11.5 系统公告（管理员）
  - 发布公告（标题、内容、生效时间）
  - 公告列表管理
  - 公告置顶/取消置顶
